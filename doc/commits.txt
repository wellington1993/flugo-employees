  queria mostrar que sei trabalhar com tipagem estatica. Nessa primeira

4930404 - limpa variaveis Firebase no Vercel (remove corrupcao CRLF)
  Esse foi o commit mais importante dessa fase.
  
  O problema real: as variaveis de ambiente tinham \r\n (caractere de nova
  linha do Windows) no final do valor. O Vercel guardava literalmente
  "flugo-employees-wts\r\n" como nome do projeto. O Firebase recebia esse
  valor corrompido e nao conseguia conectar.
  
  Tive que usar a API REST do Vercel diretamente (nao o CLI) pra recriar
  as variaveis sem o \r\n. O CLI do Vercel via PowerShell tambem adicionava
  o \r\n porque o pipe do PowerShell inclui o caractere de nova linha.
  
  Depois disso o Firebase conectou normalmente.
